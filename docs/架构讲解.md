# MiniMind 模型架构讲解

> 本文是我阅读 [`model/model_minimind.py`](../model/model_minimind.py)（共 463 行）后整理的笔记，
> 逐个拆解 MiniMind 用到的现代 LLM 关键组件，并标注对应代码位置。
> MiniMind 是一个 **类 Llama 的 Decoder-only Transformer**，最小配置：hidden=512、8 层、
> 8 个注意力头、2 个 KV 头、词表 6400，约 **26M 参数**。

## 整体结构

```
input_ids
  └─ embed_tokens (nn.Embedding, 与 lm_head 权重共享)
  └─ N × MiniMindBlock:
        x = x + Attention(RMSNorm(x))          # 预归一化 + 残差
        x = x + FeedForward(RMSNorm(x))        # SwiGLU 或 MoE
  └─ RMSNorm
  └─ lm_head (nn.Linear → vocab_size)
  └─ 交叉熵损失 (shift 一位) + MoE 辅助损失
```
关键设计（对标 Llama）：**Pre-Norm、RMSNorm、RoPE、GQA、SwiGLU、无 bias、权重共享**。

---

## 1. RMSNorm（均方根归一化）
📍 代码：[`model_minimind.py:96-106`](../model/model_minimind.py#L96)

```python
def _norm(self, x):
    return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
def forward(self, x):
    return self.weight * self._norm(x.float()).type_as(x)
```

- 与 LayerNorm 的区别：**不减均值、不加 bias**，只用均方根 `sqrt(mean(x²))` 做缩放，再乘一个可学习的 `weight`。
- **为什么用它**：比 LayerNorm 少一半统计量，计算更快，且在 LLM 上效果相当（Llama 系列的标配）。
- 细节：先 `.float()` 升精度算归一化再 `type_as(x)` 转回，避免 bf16/fp16 下数值不稳。

## 2. RoPE（旋转位置编码）
📍 代码：[`precompute_freqs_cis:109`](../model/model_minimind.py#L109)、[`apply_rotary_pos_emb:131`](../model/model_minimind.py#L131)

- **思想**：不给 token 加一个"位置向量"，而是把 Q/K 向量**按位置旋转一个角度**。两个 token 的注意力分数只依赖它们的**相对位置**，天然支持长度外推。
- 实现：预计算每个位置、每个维度的 `cos/sin`（`freqs = 1/base^(2i/d)`），推理时对 Q、K 应用
  `q*cos + rotate_half(q)*sin`（`rotate_half` 把后半维取负拼到前面，等价于复数旋转）。
- **外推能力（YaRN）**：[`config 58-65`](../model/model_minimind.py#L58) 里 `inference_rope_scaling=True` 时启用 YaRN，
  通过对不同频率段做插值/外推（`beta_fast/beta_slow` 划分高低频，线性 ramp 混合），把训练时的短上下文
  扩展到更长。这是本项目一个值得注意的工程点。

## 3. GQA（分组查询注意力）
📍 代码：[`Attention:150`](../model/model_minimind.py#L150)、[`repeat_kv:140`](../model/model_minimind.py#L140)

- 配置：`num_attention_heads=8`，`num_key_value_heads=2` → **每 4 个 Query 头共享 1 组 KV**（`n_rep=4`）。
- **为什么**：KV Cache 的显存占用正比于 KV 头数。GQA 把 KV 头从 8 减到 2，**KV Cache 显存降到 1/4**，
  推理更省显存，而效果几乎不掉。是 MHA（每头独立 KV）和 MQA（所有头共享 1 组 KV）的折中。
- `repeat_kv` 在计算注意力前把 2 组 KV 复制成 8 组对齐 Query。

## 4. Flash Attention / KV Cache
📍 代码：[`Attention.forward:169-213`](../model/model_minimind.py#L169)

- **Flash Attention**：`self.flash` 为真且满足条件时走 `F.scaled_dot_product_attention`（PyTorch 2.0+ 内置），
  否则回退到手写的 `softmax(QKᵀ/√d)·V`（含因果 mask，[line 199-209](../model/model_minimind.py#L199)）。
- **KV Cache**：[`line 185-188`](../model/model_minimind.py#L185) 推理时把历史 K/V 拼接缓存，避免每生成一个 token 都重算整段序列，是自回归生成加速的核心。

## 5. SwiGLU 前馈网络
📍 代码：[`FeedForward:216`](../model/model_minimind.py#L216)

```python
down_proj( act_fn(gate_proj(x)) * up_proj(x) )   # act_fn = SiLU
```
- 三个投影 `gate/up/down`，用 **门控**（`SiLU(gate) * up`）代替传统 MLP 的单条通路，表达力更强。
- 中间维度 `intermediate_size = 8/3 * hidden` 并向上取整到 64 的倍数（[line 219-221](../model/model_minimind.py#L219)），
  这样在保持约 2× 参数量的同时对齐硬件。全部 `bias=False`。

## 6. MoE（混合专家，可选）
📍 代码：[`MoEGate:232`](../model/model_minimind.py#L232)、[`MOEFeedForward:288`](../model/model_minimind.py#L288)

- `use_moe=True` 时把每层 FFN 换成 MoE：`n_routed_experts=4` 个路由专家 + `n_shared_experts=1` 个共享专家，
  每个 token 由门控 `MoEGate` 用 softmax 打分、选 **top-2** 专家（`num_experts_per_tok=2`）处理。
- **共享专家**：所有 token 都过，负责通用能力；**路由专家**：按 token 特化，扩容量而不显著增加计算。
- **负载均衡辅助损失**（[line 266-282](../model/model_minimind.py#L266)）：防止门控只偏爱少数专家，
  `aux_loss` 惩罚专家使用不均，权重 `aux_loss_alpha=0.01`，训练时加到总 loss（见 `train_pretrain.py` 里 `loss = res.loss + res.aux_loss`）。
- 训练/推理走不同实现：训练用 `repeat_interleave` 逐专家计算；推理 `moe_infer`（[line 328](../model/model_minimind.py#L328)）
  按专家排序后批量处理，更高效。

## 7. Transformer Block（Pre-Norm + 残差）
📍 代码：[`MiniMindBlock:352`](../model/model_minimind.py#L352)

```python
x = x + self_attn(input_layernorm(x))            # 先归一化再进子层
x = x + mlp(post_attention_layernorm(x))
```
- **Pre-Norm**（归一化放在残差分支内、子层之前）比 Post-Norm 训练更稳定，是现代 LLM 的主流选择。

## 8. 输出层与损失
📍 代码：[`MiniMindForCausalLM:427`](../model/model_minimind.py#L427)

- **权重共享**（weight tying）：`embed_tokens.weight = lm_head.weight`（[line 435](../model/model_minimind.py#L435)），
  输入嵌入和输出投影共用同一张 6400×512 的表，省参数、常见于小模型。
- **损失**：把 logits 和 labels **各错开一位**（预测下一个 token），交叉熵，`ignore_index=-100` 忽略 padding
  （[line 456-459](../model/model_minimind.py#L456)）。MoE 时再加 `aux_loss`。

## 小结

MiniMind 虽小，却几乎集齐了当代主流 LLM 的关键设计：**RMSNorm + RoPE(+YaRN) + GQA + Flash Attention +
SwiGLU + Pre-Norm + 权重共享**，可选 **MoE**。读懂这 463 行，基本就理解了 Llama/Qwen 这类模型的骨架——
这也是我做这个复现项目最大的收获：**把"论文里的名词"对应到"能跑的代码"**。
