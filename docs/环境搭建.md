## 安装步骤

### 1. 创建虚拟环境
```bash
python -m venv .venv
.venv\Scripts\activate      # Windows
```

### 2. 安装 PyTorch（关键：必须装 CUDA 版，不能用 CPU 版）
> ⚠️ **踩坑**：pip 默认 / requirements.txt 里的 `torch==2.6.0` 装出来是 **CPU 版**
> （`torch.cuda.is_available()` 返回 `False`），根本用不了 GPU。必须从 PyTorch 官方
> 的 CUDA 索引安装。我的驱动 566.24 支持到 CUDA 12.7，所以选 **cu124** 版本：
```bash
pip install torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124
```
> torch cu124 主包约 2.5GB。

### 3. 安装其余依赖
```bash
pip install -r requirements.txt
```

### 4. 验证 GPU 可用
```bash
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU')"
```

## 踩坑记录

### 坑 1：装成了 CPU 版 PyTorch
- **现象**：`torch.__version__` 显示 `2.6.0+cpu`，`torch.cuda.is_available()` 为 `False`，训练只能用 CPU（极慢）。
- **原因**：直接 `pip install torch` 或走普通 PyPI 装的是 CPU 版。
- **解决**：从 CUDA 索引重装，见上面第 2 步（`--index-url .../whl/cu124`）。

### 坑 2：C 盘空间不足，缓存/临时文件塞爆系统盘
- **现象**：C 盘仅剩 ~130MB，装 2.5GB 的 CUDA torch、下载数据集时会因 C 盘空间不足失败
  （pip 的下载缓存、解压临时目录、HuggingFace/ModelScope 数据缓存**默认全在 C 盘**）。
- **解决**：把所有缓存目录**永久重定向到 D 盘**。用 PowerShell 设置用户环境变量：
  ```powershell
  $base = "D:\AI_Projects\.cache"
  [Environment]::SetEnvironmentVariable("PIP_CACHE_DIR",         "$base\pip",            "User")
  [Environment]::SetEnvironmentVariable("HF_HOME",               "$base\huggingface",    "User")
  [Environment]::SetEnvironmentVariable("HUGGINGFACE_HUB_CACHE", "$base\huggingface\hub","User")
  [Environment]::SetEnvironmentVariable("MODELSCOPE_CACHE",      "$base\modelscope",     "User")
  [Environment]::SetEnvironmentVariable("TORCH_HOME",            "$base\torch",          "User")
  ```
  安装 torch 时再临时把 `TMP`/`TEMP` 也指到 D 盘，避免解压占 C 盘：
  ```powershell
  $env:TMP="D:\AI_Projects\.cache\tmp"; $env:TEMP="D:\AI_Projects\.cache\tmp"
  ```
